# 🔥 鱼皮热点监控 (Hot Monitor) - 需求文档

## 项目背景

鱼皮作为一个AI编程博主，需要第一时间获取热点信息（如AI大模型的更新），不能依赖人工手动搜索，而是利用工具自动发现指定的热点变化、发现最新热点，并及时发送通知，走在吃瓜第一线。

## 核心需求

### 功能1：关键词监控通知
- 用户手动输入要监控的关键词（如"GPT-5"、"Claude 4"）
- 当关键词相关的内容出现时，**第一时间发送通知**
- **利用AI识别假冒内容**，避免被标题党或虚假传闻误导

### 功能2：自动搜集热点
- 每隔一段时间，自动搜集用户指定范围（如"AI编程"）内的热点
- 用户能够查看历史热点和当前热点
- 支持多信息源，避免单一信息源

### 功能3：响应式Web页面
- 产品形态为响应式兼容的Web页面
- 前端页面要**足够独特**，不要千篇一律

### 功能4：Agent Skills技能
- 封装成Agent Skills技能，能够交给其他的AI来监控和发现热点
- **先完成网页版、确保功能正常使用之后，再开发Agent Skills**

## 技术约束

1. 轻量级工具项目，采用敏捷开发方式
2. 利用 **OpenRouter** 对接AI服务
3. 数据源：**网页搜索爬虫** + **Twitter(X) API** (twitterapi.io)
4. 需同时从多个信息源获取信息，避免单一信息源

## 非功能需求

- 响应式设计，兼容手机/平板/桌面
- 通知及时性：关键词匹配后秒级通知
- 频率控制：避免被数据源封禁
- AI准确性：能有效识别假冒/误导性内容
