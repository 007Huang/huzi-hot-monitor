# 🔥 鱼皮热点监控 (Hot Monitor) - 需求文档

> 最后更新：2026-06

---

## 项目背景

鱼皮作为一个 AI 编程博主，需要第一时间获取热点信息（如 AI 大模型的更新），不能依赖人工手动搜索，而是利用工具自动发现指定的热点变化、发现最新热点，并及时发送通知，走在吃瓜第一线。

---

## 核心需求

### 功能1：关键词监控通知

- 用户手动输入要监控的关键词（如 "GPT-5"、"Claude 4"）
- 当关键词相关的内容出现时，**第一时间发送通知**
- **利用 AI 识别假冒内容**，避免被标题党或虚假传闻误导

#### 功能细化

| 子功能 | 说明 | 状态 |
|--------|------|------|
| 添加关键词 | 输入关键词，按 Enter 添加到监控列表 | ✅ 已实现 |
| 删除关键词 | 点击关键词右侧按钮删除 | ✅ 已实现 |
| 启停关键词 | 切换关键词的 active 状态 | ✅ 已实现 |
| AI 语义匹配 | 不仅字符串匹配，理解语义（"GPT" 匹配 "GPT-5发布" 但不匹配 "GPT格式化硬盘"） | ✅ 已实现 |
| AI 假内容检测 | 识别标题党、虚假传闻、翻译错误、过时信息 | ✅ 已实现 |
| 浏览器通知 | 匹配成功后弹出浏览器通知 | ✅ 已实现 |
| 邮件通知 | 匹配成功后发送邮件（需配置 Resend） | ✅ 已实现 |
| 通知记录 | 查看历史通知日志 | ✅ 已实现 |

### 功能2：自动搜集热点

- 每隔一段时间，自动搜集用户指定范围（如 "AI编程"）内的热点
- 用户能够查看历史热点和当前热点
- 支持多信息源，避免单一信息源

#### 功能细化

| 子功能 | 说明 | 状态 |
|--------|------|------|
| 多数据源 | Bing 搜索 + DuckDuckGo + TechCrunch + Twitter | ✅ 已实现 |
| 数据源切换 | Tab 切换：全部 / 网页搜索 / Twitter | ✅ 已实现 |
| 热度评分 | 0-100 分，按评分排序展示 | ✅ 已实现 |
| 自动轮询 | 默认 5 分钟自动刷新 | ✅ 已实现 |
| AI 摘要 | 自动生成热点摘要 + 热门话题关键词 | ✅ 已实现 |
| 标签提取 | AI 自动提取 3-5 个标签 | ✅ 已实现 |
| 滚动头条 | 顶部 Marquee 展示 Top 15 热点 | ✅ 已实现 |

### 功能3：响应式 Web 页面

- 产品形态为响应式兼容的 Web 页面
- 前端页面要**足够独特**，不要千篇一律

#### 功能细化

| 子功能 | 说明 | 状态 |
|--------|------|------|
| 暗色主题 | 深色监控中心风格 | ✅ 已实现 |

| 响应式布局 | Bento Grid 三列/两列/单列自适应 | ✅ 已实现 |
| 热点详情弹窗 | 点击热点卡片查看完整信息 + AI 验证 | ✅ 已实现 |
| 假内容标记 | 虚假内容红色标记 + 半透明显示 | ✅ 已实现 |

### 功能4：Agent Skills 技能

- 封装成 Agent Skills 技能，能够交给其他的 AI 来监控和发现热点
- **先完成网页版、确保功能正常使用之后，再开发 Agent Skills**

---

## 技术约束

1. 轻量级工具项目，采用敏捷开发方式
2. 利用 **Acedata.cloud** 对接 AI 服务（DeepSeek、GPT-5.4、Claude Opus 4.8）
3. 数据源：**网页搜索爬虫** + **Twitter(X) API** (twitterapi.io)
4. 需同时从多个信息源获取信息，避免单一信息源

---

## 非功能需求

| 需求 | 说明 | 状态 |
|------|------|------|
| 响应式设计 | 兼容手机/平板/桌面 | ✅ |
| 通知及时性 | 关键词匹配后秒级通知 | ✅ |
| 频率控制 | 避免被数据源封禁（5分钟缓存 + User-Agent 轮换） | ✅ |
| AI 准确性 | 能有效识别假冒/误导性内容 | ✅ |
| 降级容错 | AI 失败时自动降级（字符串匹配/默认值） | ✅ |
| 无需数据库 | 使用 localStorage 持久化 | ✅ |

## 数据流全景

```
用户添加关键词 (localStorage)
        │
        ▼
  [useMonitor Hook] ──── 轮询(5min) ────→ GET /api/trends
        │                                      │
        │                          ┌───────────┴───────────┐
        │                          ▼                       ▼
        │                   web-search.ts            twitter.ts
        │                  ┌────┼────┐                  │
        │                  ▼    ▼    ▼                  ▼
        │               Bing  DDG  TechCrunch     twitterapi.io
        │                  └────┼────┘                  │
        │                       ▼                       │
        │                  合并 + 去重 + 评分排序 ◄──────┘
        │
        ├──→ POST /api/ai/summarize ──→ Acedata AI ──→ 摘要
        │
        ├──→ POST /api/ai/match ──→ Acedata AI ──→ 匹配？
        │         │
        │    匹配成功:
        │    ├── 浏览器通知 (Notification API)
        │    ├── POST /api/notify ──→ Resend 邮件
        │    └── 记录到 localStorage
        │
        └──→ POST /api/ai/detect ──→ Acedata AI ──→ 虚假？
```

## 待优化项

| 项目 | 说明 | 优先级 |
|------|------|--------|
| 微博数据源 | 目前仅支持 Twitter + 网页搜索，未接入微博 | 中 |
| 数据库持久化 | 当前 localStorage 在清缓存后丢失 | 低 |
| 用户认证 | 目前无登录系统 | 低 |
| 亮色主题 | 仅暗色模式，未实现主题切换 | 低 |
| 评分准确性 | 网页搜索评分含随机数，非确定性 | 中 |
| 爬虫稳定性 | Bing/DDG HTML 结构变化会导致解析失败 | 高 |
